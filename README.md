# نظام كاشير برمجلي (Barmagly POS)

نظام كاشير متكامل للمحلات والمطاعم، مبني باستخدام تقنيات الويب الحديثة (React, Node.js) ويعمل كتطبيق سطح مكتب باستخدام Electron.

## المتطلبات الأساسية (Prerequisites)

قبل البدء، تأكد من تثبيت البرامج التالية على جهازك:
*   [Node.js](https://nodejs.org/) (يفضل الإصدار 16 أو أحدث)
*   [Git](https://git-scm.com/)

## تثبيت المشروع (Installation)

1.  قم بتحميل المشروع أو استنساخه من المستودع:
    ```bash
    git clone https://github.com/khaled312001/cashier.git
    ```
2.  افتح موجه الأوامر (Terminal) داخل مجلد المشروع.
3.  قم بتثبيت الحزم والمكتبات اللازمة باستخدام الأمر:
    ```bash
    npm install
    ```

## تشغيل المشروع (Running the App)

يوجد وضعين لتشغيل المشروع: وضع الموقع الإلكتروني (Web) ووضع تطبيق سطح المكتب (Desktop App).

### 1. تشغيل كصفحة ويب (Web Development)
لتشغيل المشروع في المتصفح وتطوير الواجهة الأمامية والخلفية:
```bash
npm run dev
```
*   سيقوم هذا الأمر بتشغيل السيرفر والواجهة الأمامية.
*   يمكنك الوصول للتطبيق عبر الرابط الذي سيظهر في التيرمينال (غالبًا `http://localhost:5173`).

### 2. تشغيل كتطبيق سطح مكتب (Desktop Development)
لاختبار التطبيق كما سيظهر للمستخدم النهائي داخل نافذة Electron:
```bash
npm run electron:dev
```
*   سيفتح نافذة التطبيق مباشرة.

## بناء التطبيق (Building the Application)

لتحويل المشروع إلى ملف تنفيذي (`.exe`) قابل للتثبيت على ويندوز:

1.  تأكد من أن كل شيء يعمل بشكل صحيح.
2.  قم بتنفيذ الأمر التالي:
    ```bash
    npm run electron:build
    ```
3.  عند انتهاء العملية، ستجد ملفات التثبيت في مجلد:
    `release/`

    *   الملف سيكون بلاحقة `.exe` (مثل `نظام كاشير برمجلي Setup 1.0.0.exe`).
    *   يمكنك نسخ هذا الملف وتنصيبه على أي جهاز عميل.

## هيكلية المشروع

*   `src/`: ملفات الواجهة الأمامية (React).
*   `server/`: ملفات الخادم وقواعد البيانات المحلية (NeDB).
*   `electron/`: إعدادات تطبيق سطح المكتب.
*   `data/`: مجلد تخزين قواعد البيانات (يتم إنشاؤه تلقائياً عند التشغيل).

## حل مشاكل شائعة

*   **خطأ في التثبيت**: تأكد من أن إصدار Node.js محدث.
*   **مشكلة في الحجم عند الرفع**: تم إعداد `.gitignore` لتجاهل مجلدات `release` و `node_modules` لتقليل حجم المشروع عند الرفع على GitHub.
